# PROJECT_CONTEXT.md

## 1. 项目名称

AI短剧制作平台

## 2. 项目定位

本项目是一个面向短剧内容团队的 AI短剧制作平台 Demo，目标是模拟真实短剧生产和增长流程。

它不是单一聊天机器人，也不是简单 AI 文案工具，而是把 AI 能力嵌入短剧业务流程，形成从项目立项、内容生产、分镜生成、本地化、素材制作到增长复盘的完整链路。

## 3. 为什么做这个项目

用于面试展示：

1. 有 6 年前端开发经验，能做复杂业务系统。
2. 有 Vue3 + TypeScript + Naive UI 的前端工程能力。
3. 有 FastAPI + SQLAlchemy + SQLite 的后端接口能力。
4. 能把 AI 能力接入真实业务场景。
5. 理解 AI 应用不是简单调用接口，而是需要结构化输出、业务流程、数据沉淀和工程化兜底。
6. 能使用 Codex 等 AI 编程工具，但通过 AGENTS.md / SKILLS.md / CHECKLIST.md 约束质量和方向。

## 9. 面试讲解重点

可以这样介绍：

这个项目是一个 AI短剧制作平台，我把它设计成一个完整业务系统，而不是单点 AI 工具。系统以短剧项目 projectId 为主线，把内容策划、剧本打磨、AI分镜、多语种本地化、广告素材、媒体资产和增长分析串联起来。

前端使用 Vue3 + TypeScript + Naive UI，负责业务表单、列表管理、AI结果展示和数据可视化。后端使用 FastAPI + SQLAlchemy + SQLite，负责接口编排、数据持久化、AI服务封装和媒体上传流程。

AI 部分支持 mock 和 DeepSeek 切换，保证没有真实 Key 时也能稳定演示。媒体部分支持 mock 和 S3 presigned upload 切换，后续可以扩展真实视频转码、任务队列、多租户和权限管理。

## 10. 面试需要避免的说法

不要说：

- 这是几个 AI 页面
- 只是调用了一下大模型接口
- 都是 Codex 自动写的
- 现在还没想好业务流程
- 没有真实数据就不能演示

推荐说：

- 我用 AI 工具提升开发效率，但业务设计、模块拆分、联调和问题排查是我主导的
- 这个项目重点是 AI 能力在业务系统里的落地
- mock 是为了保证面试和本地环境稳定演示
- 真实生产环境可以替换成 DeepSeek、S3、视频转码和任务队列
- projectId 是后续数据隔离和生产链路管理的核心
